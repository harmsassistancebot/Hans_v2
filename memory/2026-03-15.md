## PKB → LLM Context Strategies Discussed

Jan asked about best ways to make vault knowledge available in Claude Code sessions.
Options evaluated:

1. **CLAUDE.md Injection** — generate from vault before session, low effort, manual trigger
2. **Symlink-Layer** — live mount vault folder into project, always current, unselective
3. **Context Recipes** — YAML-defined note bundles per use case, curated, semi-automated
4. **MCP Server** — expose vault via Model Context Protocol, full bidirectional access, ~2h setup
5. **OpenClaw on-demand extraction** — I extract + inject on request (already partially done)
6. **pkb-work/pkb-staging as live context repo** — sync hook from Obsidian save to git repo
7. **Vault-as-system-prompt** — compressed briefing injected as first message

Recommendation: CLAUDE.md injection as quick start; MCP or OpenClaw extraction long-term.
