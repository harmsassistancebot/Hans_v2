## PKB (Personal Knowledge Base) Design

### Artikel-Granularität
- Kurze Artikel zu einem Thema, ~1-2 Seiten
- Beispiele: Investmentstrategie, Buchrezension, Kontaktprofil, Projektbeschreibung

### Tag-Taxonomie (3 Dimensionen)
**type/**: profile, strategy, analysis, plan, review, reference, concept
**topic/**: technology, finance, career, geopolitics, people, health, home, photography
**status/**: draft, stable, archived
- Topics auf "Bücherregal-Ebene" halten, splitten erst ab ~30 Artikeln
- Frontmatter-Tags (YAML) für Artikel bevorzugt, kein # nötig

### Architektur-Entscheidungen
- Vault → read-only Kopie (rsync) → Extractor → LLM Task Context
- Kein Continuous Sync nötig, täglich oder on-demand
- AI hat nur Lesezugriff auf Kopie, nie auf primären Vault
- Staging-Bereich: tyunkeow/pkb_staging auf GitHub
